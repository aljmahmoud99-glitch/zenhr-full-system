STEP 2 — DATABASE
═══════════════════════════════════
Ensure these tables exist (create via migration if missing, do not recreate if already there):

job_descriptions:
  id, company_id (FK), org_node_id (nullable FK → org_nodes),
  title_ar (NOT NULL), title_en (NOT NULL), grade (varchar, e.g. G1–G10),
  min_salary (decimal, nullable), max_salary (decimal, nullable),
  responsibilities (TEXT — store as JSON array of strings),
  requirements (TEXT — JSON array),
  skills (TEXT — JSON array),
  qualifications (TEXT — JSON array),
  is_active (bool, default true),
  created_at, updated_at

career_paths:
  id, company_id (FK),
  from_job_description_id (FK → job_descriptions, RESTRICT delete),
  to_job_description_id (FK → job_descriptions, RESTRICT delete),
  min_months_required (int, default 12),
  notes (TEXT, nullable)
  UNIQUE constraint on (from_job_description_id, to_job_description_id)

Validation:
  Before creating career_path: verify both job_descriptions belong to the same company_id.
  Before deleting job_description: check no career_paths reference it AND no employees reference it.
  Return 409 Conflict with clear Arabic + English message if blocked.

Implement all endpoints. If they exist but are broken, fix them:

Job Descriptions:
  GET    /api/job-descriptions           → all for company, optional ?is_active=true filter

  POST   /api/job-descriptions           → create [hradmin] — validate title_ar + title_en required
  PUT    /api/job-descriptions/:id       → update [hradmin] — partial update allowed
  DELETE /api/job-descriptions/:id       → [hradmin] — check constraints before delete

Career Paths:
  GET    /api/career-paths               → all for company, include job title data for both ends
  POST   /api/career-paths               → create [hradmin] — validate uniqueness and company match
  DELETE /api/career-paths/:id           → [hradmin] — hard delete allowed

Response format for all endpoints must be consistent:
  Success: { success: true, data: {...} }
  Error:   { success: false, message: "..." }
  List:    { success: true, data: [...] }

All list endpoints must support:
  - Pagination: ?page=1&pageSize=20
  - Search: ?q=search_term (searches title_ar + title_en)
  - Sort: ?sortBy=grade&sortDir=asc
