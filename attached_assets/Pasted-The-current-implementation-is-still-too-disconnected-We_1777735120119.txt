- Platform Admin companies
- Org structure
- Roles
- User roles

- Employee dashboard

But the org structure is NOT reflected across the system properly.

We need a real connected multi-tenant testable system, not static/mock screens.

════════════════════════════
MAIN GOAL
════════════════════════════

Make company creation, org structure, users, roles, dashboards, and data visibility connected end-to-end.

System Admin should be able to create a company properly.
HR Admin should manage only that company.
Managers and employees should see data based on their real branch/department/team.

════════════════════════════
1. PLATFORM ADMIN — COMPANY CREATION
════════════════════════════

The System Admin companies screen must allow adding a full company/tenant.

Add/Edit Company form must include:

Company basic info:
- Company Arabic name
- Company English name
- Company code
- Commercial registration number
- Tax number
- Email
- Phone
- Website
- Country
- City
- Address
- Status active/inactive

Subscription info:
- Plan name
- Subscription start date
- Subscription end date
- Max users
- Max employees
- Trial or paid

Initial structure setup:
- Number of branches
- Branch names Arabic/English
- Branch code
- Branch location/address
- Branch manager if available

When company is created:
- Create company record
- Create root org_node for the company
- Create branch org_nodes under company
- Create default departments under branches if selected
- Create default roles for that company:
  hradmin, hr, manager, payrolladmin, employee
- Create default permissions for that company
- Create initial HR Admin user for that company if provided

Important:
Company creation must not be just UI.
It must create real backend/database records connected to org_nodes.

════════════════════════════
5. REFLECT ORG STRUCTURE ACROSS ALL SCREENS
════════════════════════════

The org structure must affect:

Dashboard:
- System Admin sees all companies
- HR Admin sees only company data
- Manager sees only team data
- Payroll sees payroll by branch/department
- Employee sees own profile/org info

Employees screen:
- HR Admin can filter by branch/department/org node
- Manager sees only team employees
- Employee does not see all employees

Documents screen:
- HR Admin sees company documents
- Manager sees team documents if permitted
- Employee sees own documents only

Payroll screen:
- Payroll Admin sees payroll by allowed branches/departments
- HR Admin sees company payroll if permission exists
- Employee sees own payslip only

Reports screen:
- Reports must have org filters:
  company, branch, department, employee
- Export/print should respect permissions and data scope

Job Descriptions:
- Job descriptions should connect to org node / department / career path
- Not isolated static UI

Attendance / Leave / Overtime:
- Must respect org scope
- Manager sees team
- Employee sees own
- HR Admin sees company
