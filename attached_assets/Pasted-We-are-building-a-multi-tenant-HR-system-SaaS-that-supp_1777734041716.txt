We are building a multi-tenant HR system (SaaS) that supports multiple companies.

This is NOT a single-company system.

═══════════════════════════════════
CORE CONCEPT — MULTI TENANCY
═══════════════════════════════════

There are TWO completely different admin types:

1) System Admin (Platform Level)
2) HR Admin (Company Level)

These MUST NOT be mixed.

───────────────────────────────────
System Admin (Platform Admin)
───────────────────────────────────
- Works at platform level (NOT tied to one company)
- Can see ALL companies in the system
- Can manage companies (tenants)

Must have a separate section:
"Platform Administration"

Includes:
- Companies list
- Company details:
  - Company name
  - Number of branches
  - Number of employees
  - Active users
  - Subscription/plan (if exists)
  - Status (active/inactive)
  - Created date
- High-level statistics:
  - Total companies
  - Total employees across system
  - Total users
- Platform-level settings

IMPORTANT:
- System Admin MUST NOT use HR screens
- Must NOT be limited to company_id

───────────────────────────────────
HR Admin (Company Admin)
───────────────────────────────────
- Belongs to ONE company only
- Can manage ONLY their company

Must have section:
"Company Administration"

Includes:
- Organizational Structure
- Roles & Permissions
- Assign Roles to Users
- Company Users

HR Admin can:
- Create branches, departments, units
- Manage employees
- Manage roles & permissions INSIDE company

IMPORTANT:
- MUST be restricted by company_id everywhere
- MUST NOT see other companies

═══════════════════════════════════
ORGANIZATIONAL STRUCTURE (ALREADY IMPLEMENTED)
═══════════════════════════════════

We have:
org_nodes table (tree structure)

Hierarchy:
Company → Branch → Department → Section → Unit

Each employee is linked to:
- company_id
- org_node_id
- (legacy department_id remains)

This structure MUST now drive the entire system.

═══════════════════════════════════
ROLE-BASED DATA VISIBILITY
═══════════════════════════════════

Different users see different data:

1) HR Admin:
- Sees FULL company data
- Can filter by branch/department
- Sees:
  employees, payroll, attendance, reports

2) Manager:
- Sees ONLY their team (based on org_node)
- Sees:
  team employees
  team attendance
  team leave requests
  team summaries

3) Payroll Admin:
- Sees payroll data ONLY
- Can filter by:
  branch / department
- Sees:
  salaries, incentives, deductions

4) Employee:
- Sees ONLY personal data
- Sees:
  own profile
  own branch/department
  direct manager
  own attendance, leave, payslips
