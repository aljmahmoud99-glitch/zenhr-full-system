═══════════════════════════════════
TASK 2 — CHAIN VALIDATION FOR EACH WIDGET
═══════════════════════════════════
For each widget/section on the employee dashboard, validate the full chain:

Widget 1: Today's Attendance Status
  1. What API is called? (GET /api/attendance/my-today or similar)
  2. What does the API return? (status code + response shape)
  3. Does the UI correctly display: present/absent/late/on_leave?
  4. Does it show clock-in time if clocked in?
  5. Scope: does the API return ONLY this employee's record? No other employee data?

Widget 2: Leave Balance Summary
  1. What API is called? (GET /api/leave/balances or /api/leave/balances/my)
  2. Response: array of { leaveType, totalDays, usedDays, remainingDays }?
  3. Does the UI correctly display remaining days per leave type?
  4. Scope: does it return ONLY this employee's balances?

Widget 3: Pending Requests Count / List
  1. What API is called for pending leave? (GET /api/leave/requests?status=pending)
  2. What API for pending overtime?
  3. What API for pending advances?
  4. Are the counts correct? (compare to DB)
  5. Scope: only own pending requests visible?

Widget 4: Latest Payslip
  1. What API is called? (GET /api/payroll/slips/my?limit=1 or similar)
  2. Does it return the most recent payslip? (check against DB)
  3. Does it show: period (month/year), net salary amount?
  4. Scope: strictly own payslip only?

Widget 5: Quick Action Buttons
  1. What buttons are visible? (Request Leave, Request Overtime, etc.)
  2. Do the buttons navigate to correct routes?
  3. Are any buttons visible that should NOT be for employee role?
