TASK 4 — PAYROLL ENGINE INTEGRATION TEST:
Goal: prove components affect payroll, not just UI.

Steps:
1. Create or ensure QA_BONUS_FIXED exists and active.
2. Assign it to employee01 if the system supports employee-specific component assignment.
   If no assignment API exists, report MISSING_ASSIGNMENT_API.
3. Create another component:
   code: QA_DEDUCT_FIXED
   nameAr: خصم اختبار ثابت
   nameEn: QA Fixed Deduction
   type: deduction
   calculationMethod: fixed
   defaultValue: 10
   taxable: false
   sscIncluded: false
   recurring: true
   active: true
   sortOrder: 901
4. Assign QA_DEDUCT_FIXED to employee01 if assignment exists.
5. Create payroll run for a future month not used before.
6. Calculate payroll.
7. Inspect employee01 payslip.
8. Expected:
   - QA_BONUS_FIXED increases gross or earnings by 25 JOD.
   - QA_DEDUCT_FIXED reduces net by 10 JOD.
   - Components appear in payslip breakdown/snapshot.
   - employee02 payslip is NOT affected unless component is global by design.

If the payroll engine ignores these components, classify:
PAYROLL_INTEGRATION_MISSING.

Fix by wiring salary component definitions / employee salary components into payroll calculation safely.

IMPORTANT:
- Do not break existing BASIC, HOUSING, OVERTIME, SSC, TAX, ADVANCE logic.
- Do not double-count basic salary.
- Only active recurring components should apply.
- Deduction components reduce net salary.
- Earning components increase gross salary.
- Taxable earnings should feed taxable income if current tax system supports that.
- SSC included earnings should feed SSC base if current SSC system supports that.
