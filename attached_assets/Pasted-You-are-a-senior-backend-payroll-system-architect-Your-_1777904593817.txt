STEP 3 — API ENDPOINTS
────────────────────────

Implement:

POST /api/payroll/runs
→ create run (draft)
→ prevent duplicate for same (month + year)

POST /api/payroll/runs/:id/calculate
→ generate slips for all employees
→ status → calculated

POST /api/payroll/runs/:id/approve
→ HR only
→ status → approved

POST /api/payroll/runs/:id/publish
→ makes slips visible to employees
→ status → published

GET /api/payroll/runs
→ list runs (role scoped)

GET /api/payroll/runs/:id/slips
→ HR: all
→ manager: team
→ employee: forbidden

GET /api/payroll/slips/me
→ employee sees only their slips
