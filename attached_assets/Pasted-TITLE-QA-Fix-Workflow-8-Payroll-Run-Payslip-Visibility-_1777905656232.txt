TASK 2 — CREATE RUN
Login as payroll.

Create payroll run for a NEW month/year not already used.

Expected:
- POST returns 201/200.
- payroll_runs row created.
- status = draft.
- employee_count = 0 before calculation.
- totals = 0 before calculation.
- duplicate run for same month/year must be blocked.

Bug types:
- FAKE_SUCCESS
- DUPLICATE_ALLOWED
- DB_INTEGRITY
- API_ERROR

TASK 3 — CALCULATE RUN
Call calculate for the draft run.

Expected:
- status becomes calculated.
- payroll_slips created for active employees.
- employee_count equals active employees count.
- total_gross > 0 if employees have salary components.
- total_net calculated.
- total_deductions calculated.
- SSC employee/employer totals calculated.
- income tax total calculated if applicable.
- approved salary advances included as advance_deduction.
- pending/rejected advances NOT deducted.
- overtime included only if approved and payment type.

Check DB:
- payroll_slips.run_id = run id.
- each slip has employee_id.
- gross_salary, deductions, net_salary are DECIMAL-safe.
- advance_deduction exists and correct.
- payslip values are snapshots, not live references.

Bug types:
- CALCULATION_ERROR
- ADVANCE_NOT_DEDUCTED
- PENDING_ADVANCE_DEDUCTED
- SLIP_MISSING
- SNAPSHOT_MISSING

TASK 4 — STATE GUARDS
Test invalid transitions:

1. Approve draft before calculate.
Expected: 400/409 blocked.

2. Publish draft before calculate/approve.
Expected: blocked.

3. Publish calculated before approve.
Expected: blocked.

4. Recalculate calculated run.
Expected: either idempotent and no duplicate slips OR blocked clearly.

5. Approve calculated run.
Expected: status = approved.

6. Publish approved run.
Expected: status = published.

7. Publish again.
Expected: blocked or idempotent without double side effects.

8. Approve after publish.
Expected: blocked.

Bug types:
- STATE_BYPASS
- DOUBLE_PUBLISH
- DUPLICATE_SLIPS
- IMMUTABILITY_BROKEN

TASK 5 — ADVANCE DEDUCTION SIDE EFFECTS
Before publish:
- Record salary_advances.remaining_balance for approved advances.

After publish:
- remaining_balance reduced exactly once.
- If balance reaches 0, status becomes settled.
- If partial balance remains, status stays approved/active.
- Publishing twice must NOT deduct again.

Bug types:
- DOUBLE_DEDUCTION
- ADVANCE_BALANCE_WRONG
- SETTLEMENT_WRONG

TASK 6 — PAYSLIP VISIBILITY / SCOPE
Login as employee01:
- GET /api/payroll/slips/my
Expected:
- employee01 sees only own slips.
- Published slips visible.
- Draft/calculated/approved unpublished slips should NOT be visible to employee unless policy says otherwise.
- No employee02 slip appears.

Login as employee02:
- Confirm employee01 slip not visible.

Login as manager.hr:
- Try GET /api/payroll/slips
- Try opening payroll run route.
Expected:
- manager cannot see payroll data unless explicitly permitted.
- No salary data leak.

Login as hr:
- HR can see company payroll if policy allows.
- HR cannot publish if only payrolladmin is allowed, unless intended.

Login as payroll:
- Payroll can see all company slips/runs.

Bug types:
- SCOPE_BUG
- ROLE_LEAK
- UNPUBLISHED_VISIBLE
- SALARY_DATA_LEAK

TASK 7 — PAYSLIP CONTENT

Expected fields:
- employee name/code
- company name
- month/year
- gross salary
- earnings breakdown
- SSC employee
- income tax
- advance deduction
- other deductions
- net salary
- run status published
- generated/approved/published date
- all money values displayed with 3 decimals
- western digits only, no Arabic-Indic numerals

Bug types:
- PAYSLIP_FIELD_MISSING
- FORMAT_ERROR
- WRONG_NET
- RTL_DISPLAY_BUG
