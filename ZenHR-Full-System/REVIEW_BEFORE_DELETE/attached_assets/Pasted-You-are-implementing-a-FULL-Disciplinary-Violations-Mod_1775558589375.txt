You are implementing a FULL Disciplinary & Violations Module in ZenJO HRMS.

This is a core HR compliance module.
It must be legally structured, traceable, and connected to employee lifecycle.

Do NOT build a simple form.
Build a complete case management system.

PRIMARY GOAL
═══════════════════════════════════════════════════════

Create a Disciplinary System where HR can:

* Record violations (cases)
* Track investigation
* Apply progressive disciplinary actions
* Link violations to employee history
* Ensure legal compliance and documentation

Store in DB (NOT hardcoded)

═══════════════════════════════════════════════════════
5) DISCIPLINARY LEVELS (PROGRESSIVE SYSTEM)
═══════════════════════════════════════════════════════

Implement progressive penalties:

1. Verbal Warning (تنبيه شفهي)
2. First Written Warning (إنذار خطي أول)
3. Second Written Warning (إنذار خطي ثاني)
4. Final Warning (إنذار نهائي)
5. Termination (فصل)

System MUST:

* Track employee disciplinary history
* Know previous warnings for same employee
* Suggest next level automatically
* Prevent skipping levels unless HR overrides

Example:
If employee already has first warning → suggest second

═══════════════════════════════════════════════════════
6) INVESTIGATION MODULE
═══════════════════════════════════════════════════════

Each case must include investigation section:

* HR Notes
* Employee Statement
* Manager Statement
* Investigation Date
* Attach evidence

This must be editable before final decision.

═══════════════════════════════════════════════════════
7) DECISION SECTION
═══════════════════════════════════════════════════════

After investigation, HR records decision:

* Action Type (warning level / termination)
* Action Date
* Salary Deduction (optional)
* Temporary Suspension (optional)
* Notes

System must:

* Link decision to disciplinary level
* Close case after decision
