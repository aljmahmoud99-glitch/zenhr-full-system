You are implementing a FULL Attendance Management Module in ZenJO HRMS.

This is one of the most critical modules in the system.
It must be accurate, scalable, and tightly integrated with shifts, leave, payroll, and disciplinary actions.

Do NOT build a simple attendance table.
Build a complete attendance operations system.

═══════════════════════════════════════════════════════
PRIMARY GOAL
═══════════════════════════════════════════════════════

Create an Attendance module where HR can:

* monitor daily attendance status
* review attendance logs
* manage attendance methods
* apply late / absence rules
* calculate overtime
* handle attendance-related requests
* generate attendance reports
* support mobile attendance with GPS and identity verification

5) LATE RULES
═══════════════════════════════════════════════════════

Attendance module must support late rules:

* grace period in minutes
* when employee becomes late
* repeated lateness tracking
* optional automatic deduction policy
* integration with disciplinary module

Examples:

* after X minutes = late
* after 3 late records = alert / disciplinary suggestion

Do NOT hardcode rules in UI.
Use configurable settings.

6) ABSENCE RULES
═══════════════════════════════════════════════════════

System must distinguish automatically between:

* absence without permission
* justified absence
* leave-covered absence

Attendance logic must check:

* approved leave
* holidays
* scheduled shifts

Absence classification must be accurate.

═══════════════════════════════════════════════════════
7) SHIFTS INTEGRATION
═══════════════════════════════════════════════════════

Attendance must read shift rules dynamically.

For each employee:

* start time
* end time
* break
* working days
* flexible/rotating shifts if assigned

System must compare attendance against employee shift to determine:

* late
* early leave
* overtime
* absence

Do NOT duplicate shift logic inside attendance.
Attendance must use shifts as the source of truth.
