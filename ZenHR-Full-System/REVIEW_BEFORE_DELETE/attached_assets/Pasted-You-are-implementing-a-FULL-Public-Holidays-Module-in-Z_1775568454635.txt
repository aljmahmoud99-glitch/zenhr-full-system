You are implementing a FULL Public Holidays Module in ZenJO HRMS.

This is a core scheduling and compliance module.
It must integrate with attendance, payroll, shifts, employee calendar, and notifications.

Do NOT build a simple holiday table.
Build a complete official holidays management system.

═══════════════════════════════════════════════════════
PRIMARY GOAL
═══════════════════════════════════════════════════════

Create a Public Holidays system where HR can:

* manage official holidays
* define holiday periods and recurrence
* integrate holidays with attendance and payroll
* support holiday notifications
* generate holiday reports

═══════════════════════════════════════════════════════

Form fields:

* Holiday Name
* Date or Date Range
* Description (optional)
* Annual Recurrence (optional)
* Scope:

  * all employees
  * selected departments only

Support:

* single-day holiday
* multi-day holiday

═══════════════════════════════════════════════════════
4) ATTENDANCE INTEGRATION
═══════════════════════════════════════════════════════

This is critical.

When a day is marked as a public holiday:

* attendance records for that day should show "On Holiday"
* employees should not be required to clock in / clock out
* if employee clocks in by mistake, system should:

  * either auto-correct the record
  * or flag it as holiday attendance depending on company settings

Attendance views and dashboards must reflect holiday status correctly.

═══════════════════════════════════════════════════════
5) PAYROLL INTEGRATION
═══════════════════════════════════════════════════════

Public holidays must affect payroll logic.

Rules:

* holiday days are paid according to company policy
* if employee works on a holiday:

  * overtime / holiday work rate may be higher
  * example: 200% instead of 150%

This must be configurable.

Integrate with:

* payroll calculation
* overtime rules
* payslip display if holiday work occurred

═══════════════════════════════════════════════════════
6) REPORTS
═══════════════════════════════════════════════════════

Build reports such as:

* annual public holidays report
* comparison between public holidays and employee consumed leave
* holidays by type
* holiday work report (employees who worked during official holidays)

Reports must support:

* year filter
* type filter
* export

═══════════════════════════════════════════════════════
7) NOTIFICATIONS & ALERTS
═══════════════════════════════════════════════════════

System must generate alerts:

* reminder one week before holiday
* reminder one day before holiday
* display holiday in employee calendar / company calendar
* notify managers for shift planning

Holiday notifications should appear in:

* dashboard widgets
* employee calendar
* app/company notifications
